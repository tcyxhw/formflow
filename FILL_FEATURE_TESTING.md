# 表单填写功能测试指南

## 功能概述

已完成表单重复填写功能，支持：
- 自动加载用户的历史提交数据
- 首次填写创建新提交
- 再次填写更新已有提交
- 填写中心显示已填写标记
- **新增**: 按分类和发布人筛选表单
- **新增**: 关键词搜索表单
- **新增**: 优化的卡片布局和样式
- **新增**: 固定高度可滚动区域

## 新增功能详解

### 搜索和筛选
1. **关键词搜索**: 在搜索框输入表单名称进行实时搜索
2. **分类筛选**: 下拉选择分类（学工、教学、科研、行政、财务等）
3. **发布人筛选**: 下拉选择发布人
4. **重置按钮**: 一键清空所有筛选条件

### 布局优化
1. **固定高度**: 可填写表单区域最高 600px，超出可滚动
2. **提交记录**: 最高 500px，超出可滚动
3. **卡片设计**: 
   - 悬停效果：上移 + 阴影 + 顶部渐变条
   - 分类标签：不同颜色区分（学工-绿色、教学-蓝色等）
   - 已填写标记：绿色成功标签
4. **响应式**: 移动端自动调整为单列布局

## 技术实现

### 后端新增
- **API 端点**: `GET /api/v1/submissions/forms/{form_id}/latest`
- **服务方法**: `SubmissionService.get_latest_submission_by_user()`
- **功能**: 获取用户对指定表单的最新提交记录

### 前端更新
- **Fill.vue**: 
  - 加载表单时自动调用 `getLatestSubmission()`
  - 如果有历史提交，将数据设置为字段的 `defaultValue`
  - 提交时判断是创建还是更新
- **FillCenter.vue**: 
  - 显示"已填写过"标签
  - 按钮文字根据状态变化
- **API 客户端**: 新增 `getLatestSubmission()` 函数

## 数据流程

```
用户点击填写 
  ↓
加载表单配置 (GET /api/v1/forms/{id})
  ↓
加载历史提交 (GET /api/v1/submissions/forms/{id}/latest)
  ↓
预填充表单字段 (设置 defaultValue)
  ↓
用户修改/确认数据
  ↓
提交表单
  ├─ 有历史提交 → PUT /api/v1/submissions/{id} (更新)
  └─ 无历史提交 → POST /api/v1/submissions (创建)
```

## 注意事项

1. **表单版本变化**: 如果表单结构改变，历史数据可能不完全匹配新字段
2. **权限检查**: 只有有 FILL 权限的用户才能看到和填写表单
3. **多次提交**: 每个用户对同一表单只保留最新的一次提交
4. **草稿功能**: 提交成功后会自动删除对应的草稿
