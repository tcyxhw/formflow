# Vue3 前端开发规范

## 基本原则
- 优先遵循现有前端架构、目录结构、组件风格和状态管理方式。
- 默认使用 Vue3 的 Composition API。
- 如果项目已使用 TypeScript，则新增和修改代码应继续保持类型化。

## 组件设计
- 组件保持单一职责，避免页面组件中堆积过多业务逻辑。
- 可复用逻辑优先抽到 composables 或 services。
- 组件 props、emits、暴露的数据结构保持清晰稳定。
- 不做无必要的全局状态扩散。

## 数据与接口
- 与后端接口相关的字段名、空值语义、分页结构、错误码解释必须与后端保持一致。
- 不要在多个组件中复制同一段请求与转换逻辑，优先集中到 API 层或 composables。
- 接口改动时同步检查调用方、类型定义和页面表现。

## 页面交互
- 异步请求必须考虑 loading、empty、error 三种基本状态。
- 表单提交必须考虑校验、禁用态、防重复提交和错误提示。
- 不要引入破坏现有用户流程的交互变更，除非任务明确要求。

## 代码质量
- 保持模板、脚本、样式职责清晰。
- 避免过深嵌套和过度 watch。
- 优先使用已有基础组件、工具函数和样式约定。
- 修改后至少运行最小必要验证，例如 lint、typecheck、test、build 中相关部分。
