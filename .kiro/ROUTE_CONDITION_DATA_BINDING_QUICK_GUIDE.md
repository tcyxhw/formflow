# 路由和条件数据绑定修复 - 快速参考指南

## 问题描述

在审批流配置中，用户遇到了两个主要问题：

### 问题 1：条件数据混乱
```
场景：
1. 在节点 A 添加条件"金额 > 1000"
2. 切换到节点 B
3. 问题：节点 B 仍然显示"金额 > 1000"的条件

预期：节点 B 应该显示自己的条件，或者没有条件
```

### 问题 2：路由属性混乱
```
场景：
1. 有三个节点：开始 → 审批 → 结束
2. 修改开始→审批路由的优先级为 10
3. 问题：审批→结束路由的优先级也变成了 10

预期：每个路由应该有独立的优先级
```

## 修复方案

### 核心改进

#### 1. 路由过滤
```typescript
// 修复前：显示所有路由
routes.value  // [开始→审批, 审批→结束]

// 修复后：只显示进入当前节点的路由
relevantRoutes  // 编辑节点"审批"时，只显示[开始→审批]
```

#### 2. 路由信息显示
```
修复前：
┌─────────────────────┐
│ 路由属性            │
│ 优先级: 1           │
│ 默认路由: ✓          │
└─────────────────────┘

修复后：
┌─────────────────────┐
│ 当前路由            │
│ 从 开始节点 到 审批节点 │
├─────────────────────┤
│ 路由属性            │
│ 优先级: 1           │
│ 默认路由: ✓          │
└─────────────────────┘
```

#### 3. 条件编辑弹窗改进
```
修复前：
编辑路由条件
配置表单字段的条件规则，支持多条件组合

修复后：
编辑路由条件
从 开始节点 到 审批节点
配置表单字段的条件规则，支持多条件组合

[当前编辑的路由]
从 开始节点 到 审批节点
```

## 使用示例

### 场景 1：为不同的路由配置不同的条件

```
步骤 1：编辑"开始→审批"路由
- 点击路由列表中的"开始→审批"
- 在路由属性面板中看到"从 开始节点 到 审批节点"
- 点击"编辑条件"
- 添加条件"金额 > 1000"
- 点击"保存所有条件"

步骤 2：编辑"审批→结束"路由
- 点击路由列表中的"审批→结束"
- 在路由属性面板中看到"从 审批节点 到 结束节点"
- 点击"编辑条件"
- 添加条件"审批人 = 张三"
- 点击"保存所有条件"

结果：
- "开始→审批"路由有条件"金额 > 1000"
- "审批→结束"路由有条件"审批人 = 张三"
- 两个条件相互独立，不会混乱
```

### 场景 2：修改路由属性

```
步骤 1：修改"开始→审批"路由的优先级
- 点击路由列表中的"开始→审批"
- 在"优先级"字段中输入 10
- 系统自动保存

步骤 2：修改"审批→结束"路由的优先级
- 点击路由列表中的"审批→结束"
- 在"优先级"字段中输入 20
- 系统自动保存

结果：
- "开始→审批"路由的优先级是 10
- "审批→结束"路由的优先级是 20
- 两个路由的优先级相互独立
```

## 关键改进点

### 1. 路由信息清晰
- ✓ 路由信息横幅显示"从 X 到 Y"
- ✓ 条件编辑弹窗中明确显示当前编辑的路由
- ✓ 用户不会混淆不同的路由

### 2. 条件独立性
- ✓ 每个路由有独立的条件
- ✓ 切换节点时不会显示其他节点的条件
- ✓ 条件编辑后保存到正确的路由

### 3. 属性独立性
- ✓ 每个路由有独立的优先级
- ✓ 每个路由有独立的默认状态
- ✓ 修改一个路由的属性不会影响其他路由

```typescript
<FlowRouteInspector

  :routes="store.routes"            // 所有路由
  :current-node-key="store.selectedNodeKey"  // 当前编辑的节点
  ...
/>
```

### 路由过滤逻辑
```typescript
// FlowRouteInspector.vue
const relevantRoutes = computed(() => {
  if (!props.currentNodeKey || !props.routes) return []
  
  // 只显示进入当前节点的路由
  return props.routes.filter(route => route.to_node_key === props.currentNodeKey)
})
```

### 路由描述生成
```typescript
const getRouteDescription = (route: FlowRouteConfig): string => {
  const fromNode = props.nodes?.find(n => getNodeKey(n) === route.from_node_key)
  const toNode = props.nodes?.find(n => getNodeKey(n) === route.to_node_key)
  
  const fromName = fromNode?.name || '未知节点'
  const toName = toNode?.name || '未知节点'
  
  return `从 ${fromName} 到 ${toName}`
}
```

## 常见问题

### Q1：为什么我看不到某个路由？
**A**：路由过滤现在只显示进入当前节点的路由。如果你在编辑节点 B，只会看到进入节点 B 的路由。

### Q2：我修改了一个路由的优先级，为什么其他路由也变了？
**A**：这个问题已经修复。现在每个路由有独立的优先级，修改一个路由的优先级不会影响其他路由。

### Q3：条件编辑弹窗中的"当前编辑的路由"是什么意思？
**A**：这是为了让你清楚地知道你正在编辑哪个路由的条件。例如，"从 开始节点 到 审批节点"表示你正在编辑这个路由的条件。

### Q4：如何确保条件被保存到正确的路由？
**A**：系统会自动根据你选择的路由来保存条件。条件编辑弹窗中会显示"当前编辑的路由"，确保你知道条件会被保存到哪个路由。
