### ✅ 6.5.3 实现保存功能

**保存流程**:
1. 用户编辑节点或路由
2. 自动标记为脏状态（`dirty = true`）
3. 点击"保存草稿"按钮
4. 调用 `store.saveDraftRemote()`
5. 发送到后端 API
6. 清除脏状态（`dirty = false`）

**保存内容**:
- 节点配置（名称、类型、审批人、驳回策略等）
- 路由配置（来源、目标、优先级、条件等）
- 节点位置信息

**错误处理**:
- 加载失败时显示错误提示
- 保存失败时显示错误提示并保留编辑状态
- 验证节点配置的有效性

### ✅ 6.5.4 实现发布功能

**发布流程**:
1. 点击"发布流程"按钮
2. 打开发布对话框
3. 输入版本标签和变更说明
4. 点击"发布"确认
5. 调用 `store.publishCurrentDraft()`
6. 创建流程快照
7. 锁定流程版本

**发布对话框**:
- 版本标签输入框
- 变更说明文本框
- 警告提示（发布后无法编辑）
- 发布/取消按钮

**错误处理**:
- 发布失败时显示错误提示
- 保留编辑状态以便重试

## 关键实现细节

### 1. 状态管理集成

```typescript
const store = useFlowDraftStore()

// 自动同步以下状态
- store.nodes: 节点列表
- store.routes: 路由列表
- store.nodesGraph: 节点位置
- store.selectedNodeKey: 选中节点
- store.selectedRouteIndex: 选中路由
- store.dirty: 脏状态标记
- store.saving: 保存中状态
- store.publishing: 发布中状态
```

### 2. 事件处理流程

```
用户操作 → 事件处理器 → Store 方法 → 状态更新 → UI 重新渲染
```

### 3. 保存流程

```
编辑 → 标记脏状态 → 点击保存 → 构建 payload → 发送 API → 更新版本 → 清除脏状态
```

### 4. 发布流程

```
点击发布 → 打开对话框 → 输入信息 → 确认 → 构建 payload → 发送 API → 创建快照 → 清除脏状态
```
