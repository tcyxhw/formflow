# 快速测试指南

## 环境准备

### 后端启动
```bash
cd backend
uvicorn app.main:app --reload --port 8000
```

### 前端启动
```bash
cd my-app
npm run dev
```

## 测试场景

### 场景 1：完整的表单填写流程

**步骤**：
1. 打开浏览器访问 `http://localhost:3000`
2. 登录系统
3. 导航到 `/form/fill-center`
4. 在"可填写的表单"列表中找到一个表单
5. 点击"开始填写"按钮
6. 填写表单中的各个字段
7. 点击"提交"按钮

**预期结果**：
- ✅ 表单页面正确加载
- ✅ 所有字段都能正确显示和交互
- ✅ 提交后看到"提交成功"消息
- ✅ 自动返回填写中心页面
- ✅ 在"我的填写记录"中看到新提交的记录

**故障排查**：
- 如果看到 400 错误，检查浏览器控制台的网络请求
- 查看后端日志中的详细错误信息
- 确保所有必填字段都已填写

### 场景 2：表单重置功能

**步骤**：
1. 打开表单填写页面
2. 填写一些字段（例如：名字、邮箱等）
3. 点击"重置"按钮
4. 观察表单变化

**预期结果**：
- ✅ 看到"表单已重置"的提示消息
- ✅ 所有已填写的字段都被清空
- ✅ 表单恢复到初始状态
- ✅ 验证状态被清除（没有红色错误提示）

**故障排查**：
- 如果字段没有被清空，检查浏览器控制台是否有 JavaScript 错误
- 刷新页面后重试

### 场景 3：重新填写已提交的表单

**步骤**：
1. 在填写中心找到已填写过的表单
2. 点击"重新填写"按钮
3. 观察表单是否预填了之前的数据

**预期结果**：
- ✅ 表单加载时预填了之前提交的数据
- ✅ 用户可以修改这些数据
- ✅ 点击提交后更新成功

**故障排查**：
- 如果没有预填数据，检查后端是否正确返回了最新提交
- 查看网络请求中的 `/submissions/forms/{form_id}/latest` 响应

### 场景 4：权限检查

**步骤**：
1. 以表单创建者身份登录
2. 访问自己创建的表单的填写页面
3. 验证能否进入填写页面

**预期结果**：
- ✅ 表单创建者能够进入填写页面
- ✅ 不显示"暂无填写权限"的警告

**故障排查**：
- 如果看到权限错误，检查后端的 `/permissions/me` 端点
- 查看后端日志中的权限检查日志

## 调试技巧

### 1. 查看网络请求
```
1. 打开浏览器开发者工具（F12）
2. 切换到"网络"标签
3. 执行操作（如提交表单）
4. 查看请求和响应
5. 检查状态码和响应体
```

### 2. 查看浏览器控制台
```
1. 打开浏览器开发者工具（F12）
2. 切换到"控制台"标签
3. 查看是否有 JavaScript 错误
4. 查看自定义日志输出
```

### 3. 查看后端日志
```
1. 查看后端终端输出
2. 搜索关键词：
   - "创建提交请求" - 提交请求日志
   - "Validation error" - 验证错误
   - "Create submission error" - 提交错误
```

### 4. 使用 Swagger 文档
```
1. 访问 http://localhost:8000/api/v1/docs
2. 找到相关的 API 端点
3. 点击"Try it out"进行测试
4. 查看请求和响应
```

## 常见问题

### Q: 提交时出现 400 错误
**A**: 
- 检查表单数据是否为空
- 查看后端日志中的详细错误信息
- 确保所有必填字段都已填写
- 检查数据格式是否正确

### Q: 重置按钮没有反应
**A**:
- 检查浏览器控制台是否有 JavaScript 错误
- 刷新页面后重试
- 检查是否有浏览器扩展程序干扰

### Q: 无法进入表单填写页面
**A**:
- 检查权限是否正确授予
- 查看后端的权限检查日志
- 确保表单已发布

### Q: 预填数据没有显示
**A**:
- 检查是否有之前的提交记录
- 查看网络请求中的 `/submissions/forms/{form_id}/latest` 响应
- 检查后端是否正确返回了提交数据

## 性能测试

### 1. 加载时间
- 表单填写页面应该在 2 秒内加载
- 表单列表应该在 3 秒内加载

### 2. 提交响应时间
- 表单提交应该在 5 秒内完成
- 如果超过 5 秒，检查后端是否有性能问题

### 3. 内存使用
- 打开多个表单填写页面不应该导致浏览器崩溃
- 关闭页面后内存应该被释放
