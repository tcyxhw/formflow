# 审批流程配置器 UX 缺陷修复

## 介绍

审批流程配置页面（/flow/configurator）存在三个关键缺陷，影响用户配置审批流程的能力。这些缺陷包括：条件设置交互问题、开始/结束节点显示不必要的配置选项，以及路由属性逻辑错误导致路由设置相互覆盖。这些问题严重影响了审批流程的创建和编辑功能。

## 缺陷分析

### 当前行为（缺陷）

1.1 当用户点击"添加条件"按钮时，系统在下方显示一行条件配置界面，但该界面应该在弹窗中打开以提供更好的编辑体验

1.2 当用户在开始节点上进行操作时，系统在右下角显示条件设置和路由属性配置选项，但开始节点不需要这些配置

1.3 当用户在结束节点上进行操作时，系统在右下角显示条件设置和路由属性配置选项，但结束节点不需要这些配置

1.4 当用户为审批节点设置路由属性（源节点为开始节点，目标节点为审批节点）后，再为结束节点设置路由属性时，系统会取消掉之前设置的开始节点到审批节点的路由连接

1.5 当用户设置多个节点的路由属性时，后设置的路由会覆盖之前设置的路由，导致只有最后一条路由被保留

### 期望行为（正确）

2.1 当用户点击"添加条件"按钮时，系统应该打开一个弹窗，允许用户在弹窗中编辑审批条件

2.2 当用户在开始节点上进行操作时，系统不应该在右下角显示条件设置和路由属性配置选项

2.3 当用户在结束节点上进行操作时，系统不应该在右下角显示条件设置和路由属性配置选项

2.4 当用户为审批节点设置路由属性后，再为其他节点设置路由属性时，系统应该保留之前设置的所有路由连接

2.5 当用户设置多个节点的路由属性时，系统应该独立管理每个节点的路由配置，不同节点的路由设置应该相互独立

### 不变行为（防止回归）

3.1 当用户在普通审批节点上进行操作时，系统应该继续在右下角显示条件设置和路由属性配置选项

3.2 当用户点击其他 UI 元素时，系统应该继续正常响应，不受路由属性修复的影响

3.3 当用户保存审批流程时，系统应该继续正确保存所有节点和连接信息

3.4 当用户加载已保存的审批流程时，系统应该继续正确显示所有节点和路由连接
