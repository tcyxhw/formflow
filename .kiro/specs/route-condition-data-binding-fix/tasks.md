# 路由和条件数据绑定修复 - 任务清单

## 任务概览

本 spec 包含以下主要任务：

- [x] 1. 需求分析与设计
- [x] 2. 前端路由过滤修复
- [x] 3. 条件显示与编辑修复
- [x] 4. 路由属性绑定修复
- [x] 5. 集成测试与验证

---

## 1. 需求分析与设计

- [x] 1.1 分析 FlowRouteInspector 组件的当前实现
- [x] 1.2 分析 flowDraft 状态管理中的路由数据结构
- [x] 1.3 分析 ConditionBuilderV2 组件的条件编辑逻辑
- [x] 1.4 创建详细的设计文档

---

## 2. 前端路由过滤修复

- [x] 2.1 修复 FlowRouteInspector 组件的路由过滤逻辑
  - 当编辑节点时，只显示进入该节点的路由（to_node_key = 当前节点）
  - 添加计算属性 `relevantRoutes` 来过滤路由

- [x] 2.2 修复路由选择逻辑
  - 确保用户选择的路由是正确的
  - 在路由列表中显示路由的来源和目标节点

- [x] 2.3 编写单元测试
  - 测试路由过滤是否正确
  - 测试路由选择是否正确

---

## 3. 条件显示与编辑修复

- [x] 3.1 修复条件显示逻辑
  - 在条件构建器中显示路由的来源和目标节点
  - 明确指出这是哪个路由的条件

- [x] 3.2 修复条件编辑逻辑
  - 确保条件编辑后保存到正确的路由
  - 防止条件被错误地保存到其他路由

- [x] 3.3 修复条件清除逻辑
  - 确保清除条件时只清除当前路由的条件
  - 不影响其他路由的条件

- [x] 3.4 编写单元测试
  - 测试条件显示是否正确
  - 测试条件编辑是否正确
  - 测试条件清除是否正确

---

## 4. 路由属性绑定修复

- [x] 4.1 修复路由优先级编辑
  - 确保每个路由有独立的优先级
  - 修改优先级时只影响当前路由

- [x] 4.2 修复默认路由标记
  - 确保每个路由有独立的 is_default 标记
  - 修改默认状态时只影响当前路由

- [x] 4.3 编写单元测试
  - 测试优先级编辑是否正确
  - 测试默认状态修改是否正确

---

## 5. 集成测试与验证

- [x] 5.1 编写集成测试
  - 测试在不同节点编辑条件是否相互独立
  - 测试路由属性是否正确关联到具体的路由
  - 测试保存和加载流程是否正确

- [x] 5.2 手动测试
  - 创建多节点审批流
  - 在不同节点添加不同的条件
  - 验证条件是否相互独立

- [x] 5.3 回归测试
  - 验证基本的流程创建功能是否正常
  - 验证流程保存和加载是否正常
  - 验证节点删除是否正常
