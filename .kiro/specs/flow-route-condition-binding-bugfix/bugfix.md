# 流程审批路由条件绑定 Bug 修复

## 介绍

在表单流程审批配置页面（flow/configurator）中，路由属性和路由条件的绑定存在两个关键问题：

1. **路由属性绑定问题**：路由属性和路由条件没有与当前节点绑定，导致所有节点共享同一个路由属性和条件，而不是只显示进入该节点的路由。

2. **条件显示问题**：条件展示使用英文字段名（如 `student_id`），而不是中文字段标签（如 "学号"），降低了用户体验。

这两个问题严重影响了用户在配置复杂审批流程时的工作效率和准确性。

## Bug 分析

### Current Behavior（缺陷）

1.1 WHEN 用户在流程审批配置页面选择一个节点时，THEN 系统显示所有路由的属性和条件，而不仅仅是进入该节点的路由

1.2 WHEN 用户查看路由条件时，THEN 系统使用英文字段名（如 `student_id`）显示条件，而不是中文字段标签（如 "学号"）

1.3 WHEN 用户编辑路由条件中的字段选择时，THEN 系统显示的字段列表使用英文字段名，而不是中文字段标签

### Expected Behavior（正确）

2.1 WHEN 用户在流程审批配置页面选择一个节点时，THEN 系统应该只显示进入该节点的路由（`to_node_key` 等于当前节点）的属性和条件

2.2 WHEN 用户查看路由条件时，THEN 系统应该使用中文字段标签（从表单 Schema 中获取）显示条件，格式为"中文标签 运算符 数值"（例如"学号等于1"）

2.3 WHEN 用户编辑路由条件中的字段选择时，THEN 系统应该在字段列表中显示中文字段标签，同时保持字段键作为内部标识

### Unchanged Behavior（保持不变）

3.1 WHEN 用户编辑其他路由属性（如优先级、默认路由标志）时，THEN 系统应该继续正常保存这些属性

3.2 WHEN 用户在条件编辑模态框中添加、编辑或删除条件时，THEN 系统应该继续正常处理这些操作

3.3 WHEN 用户切换不同的节点时，THEN 系统应该正确更新显示的路由列表和条件

3.4 WHEN 用户查看 JSON 编辑器中的 JsonLogic 格式时，THEN 系统应该继续使用英文字段名作为内部表示（不改变数据格式）
