# 需求文档：表单权限系统

## 1. 概述

### 1.1 功能概述
实现高校多租户表单平台的权限管理系统，支持基于用户、角色、部门、岗位的多维度权限控制。

### 1.2 核心特性
- 用户-部门多对多关系
- 部门层级权限继承
- 表单创建者自动拥有管理权限
- 批量授权功能
- 下拉选择器数据接口

## 2. 功能需求

### 2.1 用户-部门关联 (FR-001)
**优先级**: 高

**需求描述**:
- 创建 user_department 中间表
- 支持用户关联多个部门
- 标记主部门
- 支持部门变更历史

**验收标准**:
- [ ] user_department 表创建成功
- [ ] 用户可关联多个部门
- [ ] 可设置主部门
- [ ] 关联关系可查询

### 2.2 权限级别定义 (FR-002)
**优先级**: 高

**需求描述**:
- VIEW: 查看权限
- EDIT: 编辑权限
- DELETE: 删除权限
- MANAGE: 管理权限（最高）

**验收标准**:
- [ ] 权限级别枚举定义完整
- [ ] 权限级别可比较（VIEW < EDIT < DELETE < MANAGE）
- [ ] 权限级别在前端可选择

### 2.3 权限授予 (FR-003)
**优先级**: 高

**需求描述**:
- 支持按用户授权
- 支持按角色授权
- 支持按部门授权
- 支持按岗位授权
- 支持部门层级继承（include_children）

**验收标准**:
- [ ] 可授予用户权限
- [ ] 可授予角色权限
- [ ] 可授予部门权限
- [ ] 部门授权可设置包含子部门
- [ ] 可授予岗位权限

### 2.4 权限校验 (FR-004)
**优先级**: 高

**需求描述**:
- 表单创建者自动拥有 manage 权限
- 校验用户对表单的操作权限
- 支持部门层级继承校验
- 权限级别包含判断（高权限包含低权限）

**验收标准**:
- [ ] 创建者自动拥有管理权限
- [ ] 权限校验返回布尔值
- [ ] 部门层级继承生效
- [ ] 权限包含关系正确

### 2.5 批量授权 (FR-005)
**优先级**: 中

**需求描述**:
- 支持一次性授予多个权限
- 批量操作事务保证
- 部分失败可回滚

**验收标准**:
- [ ] 批量授权接口可用
- [ ] 全部成功或全部回滚
- [ ] 返回成功/失败详情

### 2.6 权限撤销 (FR-006)
**优先级**: 高

**需求描述**:
- 支持删除权限记录
- 权限撤销立即生效

**验收标准**:
- [ ] 可删除权限记录
- [ ] 撤销后权限立即失效

### 2.7 下拉选择器接口 (FR-007)
**优先级**: 中

**需求描述**:
- 获取可授权用户列表
- 获取可授权角色列表
- 获取可授权部门列表
- 获取可授权岗位列表

**验收标准**:
- [ ] 用户列表接口返回正确数据
- [ ] 角色列表接口返回正确数据
- [ ] 部门列表接口返回正确数据
- [ ] 岗位列表接口返回正确数据

### 2.8 权限检查接口 (FR-008)
**优先级**: 中

**需求描述**:
- 提供权限检查 API
- 返回用户拥有的权限级别

**验收标准**:
- [ ] 检查接口返回 has_permission
- [ ] 检查接口返回用户权限级别

## 3. 非功能需求

### 3.1 性能需求 (NFR-001)
- 权限校验响应时间 < 100ms
- 批量授权支持 > 100 条记录

### 3.2 安全需求 (NFR-002)
- 权限校验在服务层执行
- 禁止前端绕过权限控制
- 操作日志记录

### 3.3 多租户需求 (NFR-003)
- 权限数据按租户隔离
- 租户 ID 从请求头获取

## 4. 依赖关系

### 4.1 前置依赖
- 用户模块
- 角色模块
- 部门模块
- 岗位模块
- 表单模块

### 4.2 数据库依赖
- PostgreSQL（支持递归 CTE）
- Alembic 迁移工具
