# 审批流程配置UX缺陷修复

## 介绍

审批流程配置界面存在两个UX问题，影响用户的配置体验：

1. **条件设置空间受限**：条件配置在右侧面板中空间太小，用户难以有效配置复杂的条件表达式
2. **开始/结束节点显示不必要的配置**：开始和结束节点本质上不需要审批相关配置，但当前UI对所有节点类型显示相同的配置选项

这两个问题导致用户在配置审批流程时体验不佳，需要改进。

## Bug分析

### 当前行为（缺陷）

1.1 当用户在FlowNodeInspector右侧面板中点击"编辑条件"按钮时，条件编辑器在受限的面板空间内显示，用户难以看到完整的条件表达式和所有配置选项

1.2 当用户选择开始节点（start）进行配置时，系统显示负责人类型、审批策略、SLA、允许代理、驳回策略、自动审批等审批相关配置选项，这些对开始节点没有意义

1.3 当用户选择结束节点（end）进行配置时，系统显示负责人类型、审批策略、SLA、允许代理、驳回策略、自动审批等审批相关配置选项，这些对结束节点没有意义

### 期望行为（正确）

2.1 当用户在FlowNodeInspector中点击"编辑条件"按钮时，系统应弹出一个独立的、更大的模态框，为条件编辑提供充足的操作空间

2.2 当用户选择开始节点进行配置时，系统应仅显示基本节点信息（节点名称、节点类型），不显示任何审批相关配置选项

2.3 当用户选择结束节点进行配置时，系统应仅显示基本节点信息（节点名称、节点类型），不显示任何审批相关配置选项

### 不变行为（防止回归）

3.1 当用户选择人工审批节点（user）进行配置时，系统应继续显示所有审批相关配置选项（负责人类型、审批策略、SLA、允许代理、驳回策略、自动审批）

3.2 当用户选择自动节点（auto）进行配置时，系统应继续显示所有审批相关配置选项

3.3 当用户选择条件节点（condition）进行配置时，系统应继续显示条件分支配置，不显示审批相关配置

3.4 当用户在条件编辑模态框中编辑条件表达式时，系统应继续支持所有现有的条件构建功能（添加规则、添加分组、删除规则等）

3.5 当用户在条件编辑模态框中保存或取消时，系统应继续正确处理条件数据的保存或丢弃
