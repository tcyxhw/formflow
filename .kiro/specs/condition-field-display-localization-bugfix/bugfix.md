# 流程审批路由条件字段显示本地化修复需求文档

## 介绍

修复流程审批路由条件配置中字段显示的本地化问题。虽然之前的修复通过了测试，但在实际页面使用中仍存在字段选择器显示英文字段名、"系统字段"异常替换、以及条件显示未本地化等问题。这些问题影响了用户体验，需要确保 FieldLabelService.getFieldLabel() 被正确调用并返回中文标签。

## Bug 分析

### 当前行为（缺陷）

1.1 当用户在添加条件中选择字段时，系统显示英文字段名（如 "student_id"）而不是中文名称

1.2 当用户滑动选择字段时，系统将最上方的字段替换成"系统字段"文本

1.3 当条件数据显示时，系统使用英文字段名（如 "student_id == 1"）而不是中文标签

1.4 当 FieldLabelService.getFieldLabel() 方法存在时，系统未正确调用该方法获取中文标签

### 预期行为（正确）

2.1 当用户在添加条件中选择字段时，系统应显示中文标签而不是英文字段名

2.2 当用户滑动选择字段时，系统应保持字段选项的正确显示，不应出现"系统字段"替换

2.3 当条件数据显示时，系统应使用中文标签（如"学号 等于 1"）而不是英文字段名

2.4 当 FieldLabelService.getFieldLabel() 方法存在时，系统应正确调用该方法并使用返回的中文标签

### 不变行为（回归预防）

3.1 当字段选择器的基本功能正常时，系统应继续保持字段选择的核心功能

3.2 当条件逻辑验证正确时，系统应继续保持条件验证的准确性

3.3 当现有的字段数据结构完整时，系统应继续保持数据结构的完整性

3.4 当其他非字段显示相关的功能正常时，系统应继续保持这些功能的正常运行
