## 测试覆盖范围

### Bug 1: 路由过滤 - 只显示进入当前节点的路由
- ✗ 应该只显示进入节点 B 的路由（to_node_key = node_2）
- ✗ 不应该显示从节点 B 出发的路由

**失败原因**: 组件没有实现路由过滤逻辑，显示所有路由而不是只显示进入当前节点的路由。

### Bug 2: 条件数据关联 - 条件正确关联到具体的路由
- ✗ 编辑节点 B 时应该显示进入节点 B 的路由的条件
- ✗ 切换到不同节点时应该显示该节点的条件

**失败原因**: 条件数据没有正确关联到具体的路由，切换节点时仍然显示前一个节点的条件。

### Bug 3: 字段标签显示 - 条件显示使用中文标签
- ✗ 条件显示应该使用中文字段标签而不是英文字段名
- ✗ 条件编辑弹窗中应该显示中文标签

**失败原因**: `formatConditionForDisplay` 函数没有使用 `FieldLabelService` 获取中文标签，直接显示字段键。

### Bug 4: 条件编辑弹窗样式 - 条件项有清晰的视觉分隔
- ✗ 条件列表应该有清晰的样式和布局
- ✗ 每个条件项应该有边框、间距和操作按钮
- ✗ 条件列表应该显示条件计数

**失败原因**: 组件挂载失败（缺少 n-dialog-provider），但样式定义不完整。

### Bug 5: 路由属性独立性 - 每个路由有独立的属性值
- ✗ 不同的路由应该有独立的优先级
- ✗ 不同的路由应该有独立的默认状态

**失败原因**: 路由属性没有正确绑定到具体的路由。

### 保留需求 - 基本功能不受影响
- ✗ 应该能够正确识别路由的来源和目标节点
- ✗ 应该能够正确处理条件表达式
- ✗ 应该能够正确处理没有条件的路由

**失败原因**: 组件挂载失败，无法测试基本功能。

## 关键发现

### 1. 路由过滤缺失
组件没有实现 `relevantRoutes` 计算属性来过滤路由。当前显示所有路由，而不是只显示进入当前节点的路由。

### 2. 字段标签映射缺失
`formatConditionForDisplay` 函数没有使用 `FieldLabelService` 获取字段的中文标签。需要集成字段标签服务。

### 3. 条件编辑弹窗样式不完善
`conditions-list-section` 的样式定义不完整，条件项缺乏清晰的边框、间距和背景色。

### 4. 组件依赖问题
组件使用了 `useDialog()` 但测试环境中没有提供 `n-dialog-provider`。这需要在测试中修复。
