# 审批流路由条件配置修复

## 介绍

流程审批配置中存在三个主要问题，影响了用户的使用体验和数据正确性。路由属性和条件没有正确关联到具体的节点，导致在编辑不同节点时显示错误的条件数据；条件显示使用英文字段名而不是中文标签，降低了可读性；条件编辑弹窗的样式混乱，已添加条件的展示缺乏清晰的视觉分隔。本修复旨在确保每个路由有独立的条件和属性，使用中文字段标签显示条件，并改进条件编辑弹窗的UI样式。

## Bug 分析

### 当前行为（缺陷）

1.1 WHEN 用户在节点 A 添加条件后切换到节点 B THEN 系统仍然显示节点 A 的条件，即使节点 B 有不同的路由

1.2 WHEN 用户编辑节点 B 的路由属性 THEN 系统显示所有路由而不是只显示进入节点 B 的路由（to_node_key = B）

1.3 WHEN 用户在条件编辑弹窗中查看已添加的条件 THEN 系统显示英文字段名（如 student_id）而不是中文字段标签（如"学号"）

1.4 WHEN 用户在条件编辑弹窗中查看已添加的条件 THEN 系统没有为条件提供清晰的视觉分隔和布局，条件之间没有边框、间距和操作按钮

1.5 WHEN 用户在节点 A 修改路由优先级或默认状态 THEN 系统没有正确区分不同的路由，导致多个路由共享同一个属性值

### 期望行为（正确）

2.1 WHEN 用户在节点 A 添加条件后切换到节点 B THEN 系统只显示与节点 B 相关的路由的条件，不显示节点 A 的条件

2.2 WHEN 用户编辑节点 B 的路由属性 THEN 系统只显示进入节点 B 的路由（to_node_key = B），不显示其他路由

2.3 WHEN 用户在条件编辑弹窗中查看已添加的条件 THEN 系统只显示中文字段标签（如"学号"），不显示英文字段名（如 student_id）

2.4 WHEN 用户在条件编辑弹窗中查看已添加的条件 THEN 系统为每个条件提供清晰的视觉分隔（边框、间距）和操作按钮（编辑、删除）

2.5 WHEN 用户在节点 A 修改路由优先级或默认状态 THEN 系统正确区分不同的路由，每个路由有独立的属性值

### 未改变的行为（防止回归）

3.1 WHEN 用户创建新的审批流 THEN 系统仍然能够正确创建节点和路由，不影响基本的流程创建功能

3.2 WHEN 用户保存审批流配置 THEN 系统仍然能够正确保存所有节点和路由数据到数据库

3.3 WHEN 用户查看已保存的审批流 THEN 系统仍然能够正确加载并显示所有节点和路由数据

3.4 WHEN 用户删除节点 THEN 系统仍然能够正确删除相关的路由和条件数据

3.5 WHEN 用户在条件构建器中添加条件规则 THEN 系统仍然能够正确验证条件表达式的有效性

3.6 WHEN 用户在条件编辑弹窗中编辑条件 THEN 系统仍然能够正确保存条件到路由配置中
