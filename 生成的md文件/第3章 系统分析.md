# 3  系统分析
## 3.1  可行性分析
### 3.1.1  技术可行性
前端采用Vue 3框架与TypeScript，利用组合式API和强类型约束，能够高效构建可维护的交互式用户界面。UI组件选用Naive UI，提供统一的设计语言和丰富的交互组件，保证前端开发效率与界面一致性。在低代码能力方面，系统集成vuedraggable实现拖拽式表单设计器，并引入AntV X6图编辑引擎构建可视化流程配置器，使用户无需编写代码即可自定义表单字段布局与审批流程结构。数据可视化采用ECharts图表库，为统计看板提供折线图、柱状图、饼图等多类型图表的渲染能力。后端采用FastAPI框架，该框架基于Python异步编程模型，具备高性能请求处理能力与自动生成OpenAPI交互文档的特性，便于前后端协作与接口调试。数据持久化采用SQLAlchemy ORM与PostgreSQL关系型数据库，兼顾开发效率与数据一致性。系统引入Redis作为缓存中间件，实现JWT令牌黑名单与热数据缓存。文件存储采用MinIO兼容S3协议的对象存储服务，支持可靠的文件上传与管理。认证机制基于JWT双令牌与bcrypt加密算法，保证系统的访问安全。此外，系统集成大语言模型API，支持通过自然语言描述自动生成表单结构，进一步降低表单创建门槛。综上所述，本系统所选技术栈成熟稳定且社区活跃，系统开发在技术上是可行的。
### 3.1.2  经济可行性
本系统的开发完全由个人开发者完成，意味着开发成本较低。个人开发者具备相关技术知识和编程能力，可以直接参与系统前后端的设计与开发工作，无需雇佣其他专业人员，因此可以有效节省人力资源成本和管理成本。此外，系统开发过程中使用的FastAPI、Vue 3、PostgreSQL、Redis、MinIO等均为免费的开源工具和软件，无需支付商业许可费用，进一步降低了系统的经济负担。系统部署可依托高校现有服务器资源或云服务平台，后期运维成本可控。因此，本系统的开发在经济上是可行的。
### 3.1.3  操作可行性
系统的用户界面采用Vue 3框架与Naive UI组件库开发，确保了界面的直观性和交互友好性。对于本系统的使用者，包括超级管理员、部门岗位人员和普通用户，系统均提供清晰的操作界面与导航结构，降低了不同角色用户的学习和适应门槛。本系统的核心特色在于低代码设计理念，通过拖拽式表单设计器与可视化流程配置器，使得非技术背景的高校管理人员也能快速创建和发布表单及审批流程，极大降低了系统使用的技术门槛。此外，系统的功能模块划分明确，如首页工作台、表单管理、审批任务、活动管理等，各模块独立清晰，使用户能够快速定位所需功能。操作可行性还体现在系统的响应速度和运行稳定性上，FastAPI异步架构提供了良好的并发处理能力，这些都是提升用户操作体验的重要因素。因此，本系统的开发在操作上是可行的。
## 3.2  需求分析
### 3.2.1  系统功能需求
（1）注册与登录
低代码高校表单审批平台需要为用户提供多租户注册与登录功能。用户首次访问时需选择所属租户（学校或机构），然后在相应租户下完成注册与登录。系统应验证用户提供的登录凭据，采用JWT双令牌机制实现无状态认证，并根据用户角色分配相应的功能权限和菜单访问权限。
（2）表单设计
表单设计是本系统的核心功能，旨在为用户提供低代码、可视化的表单创建体验。该模块包含拖拽式表单设计器，支持文本、数字、日期、下拉选择、文件上传、公式计算等十余种字段类型的拖拽布局与属性配置。系统提供表单版本管理功能，支持多版本schema的保存。字段联动逻辑编辑器支持条件显示、隐藏与必填校验规则的自定义配置。计算字段引擎支持跨字段的动态数值运算。此外，模块集成AI辅助生成功能，用户通过自然语言描述即可自动生成表单初始结构，大幅缩短表单设计周期。系统还提供预置表单模板库，覆盖学生请假、活动报名、成绩申报等高校常见业务场景。
（3）表单填写与提交
该模块面向终端用户提供表单填写与数据提交功能。系统根据已发布的表单schema动态渲染填写界面，支持各类字段类型的输入与校验。提交过程支持草稿自动保存，用户可随时暂存填写进度并在后续继续完成。模块支持多附件上传，使用MinIO对象存储方式。提交完成后系统会保存表单schema快照，确保历史数据的结构完整性。系统还支持批量导入与批量提交操作，满足大规模数据采集场景的需求。
（4）审批流程管理
审批流程管理是本系统的另一核心功能，负责表单提交后的流转与审批控制。该模块提供可视化流程配置器，支持审批节点、条件分支节点、并行分支节点与自动审批节点的拖拽编排。审批人指派策略灵活多样，支持按角色、按岗位以及按指定用户分配。系统支持会签机制，可配置任一通过或全部通过的会签规则。SLA时效管理功能允许为每个审批节点配置处理时限，系统自动追踪超时任务。驳回策略支持驳回至起点或驳回至上一节点。此外，系统提供代理人机制与转办加签功能，满足审批人临时离岗或特殊情况下的流程延续需求。全链路操作日志记录审批过程中的每一步操作，保证审批轨迹的可追溯性。
（5）数据统计与看板
数据统计与看板模块负责对系统中的关键数据进行汇总与可视化展示。首页工作台展示待办任务数量、表单提交总量、审批通过率等概览数据。提交量趋势图以日维度展示表单提交量的变化趋势。
（6）系统管理
系统管理是超级管理员维护系统运行的基础模块。用户管理支持对系统内用户信息、角色及权限的增删改查操作，并支持通过Excel批量导入用户数据。组织架构管理支持学院、部门、班级等树形结构的配置与维护。审批小组管理允许管理员创建审批小组并动态调整成员。审计日志记录系统中所有关键操作，确保操作行为可追溯。字典管理与定时任务管理为系统提供灵活的数据字典配置与自动化任务调度能力。
### 3.2.2  系统应用需求
低代码高校表单审批平台的系统应用需求涉及三种主要角色：超级管理员、部门岗位人员和普通用户。根据每个角色的职责和业务目标，系统应提供不同的功能和权限以满足其需求。系统用例图如图1所示。

（1）超级管理员
超级管理员是系统的最高权限用户，负责平台的整体运维与全局管理。其需求包括：用户与组织管理，通过管理系统内的全部用户账号、部门架构、岗位信息与角色权限，构建完整的组织人员体系。表单管理让超级管理员审核、发布和管理全部表单模板，确保表单的规范性与合规性。流程管理通过配置审批流程的定义与发布，建立符合高校业务规范的审批流转机制。数据管理让超级管理员查看系统的全部提交记录、审批记录与统计数据，支持数据导出功能。活动管理通过创建和管理活动，支持课外活动的报名、签到与评奖流程。审计日志管理允许超级管理员查看系统的全部操作日志，进行安全审计。全局配置管理允许超级管理员统一管理系统的全局字典、定时任务与运行参数。
（2）部门岗位人员
部门岗位人员是系统中承担管理职责的核心角色，其功能模块与超级管理员一致，但数据的可见与操作范围受部门归属和岗位层级的约束，仅能管理同部门内岗位等级低于自身的用户及相关数据。其需求包括：用户管理，可查看和管理本部门内岗位等级低于自身的用户信息，支持通过Excel批量导入本部门用户数据。表单管理让部门岗位人员审核、发布和管理本部门范围内的表单模板。流程管理通过配置审批流程的定义与发布，建立本部门业务所需的审批流转机制。审批管理支持部门岗位人员对待审批任务进行审核与决策，可填写审批意见后通过或驳回申请，按SLA等级和任务状态进行优先级排序与处理，会签场景下可查看其他审批人的处理进展与意见，同时支持任务转办、加签以及代理人委托功能。数据管理让部门岗位人员查看本部门的提交记录、审批记录与统计数据，支持数据导出功能。活动管理通过创建和管理本部门的活动，支持课外活动的报名、签到与评奖流程。
（3）普通用户
普通用户是表单的填写者与活动的参与者，其需求包括：用户能够注册新账号并完成登录，确保个人信息安全。表单填写功能为用户提供在线的表单浏览与填写体验，支持草稿保存、附件上传与提交操作。提交记录管理让用户查看自己的提交历史与审批进展状态。活动参与功能为用户提供活动浏览、在线报名、签到打卡等完整的参与流程。学分查询功能让用户查看个人的学分明细与变动记录。证书查看与验证功能支持用户下载活动证书并通过验证码核验证书真伪。
