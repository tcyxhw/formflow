2022 年 1 月一、实习单位及岗位简介

1.实习单位简介

东莞市长泰模具热处理有限公司成立于 2010 年 07 月 20 日，注册地位于东莞市高埗镇

宝莲村宝莲二桥。经营范围包括模具热处理加工、通用机械设备加工，并提供模具材料、

五金材料的销售。具备先进的生产设备和专业的技术团队，能够为客户提供包括淬火、回

火、表面处理等在内的全方位热处理解决方案。

2.实习岗位简介

实习岗位为研发部门的助理工程师岗位，参与电池生产后段工艺的化成分容环节的上

位机与下位机系统开发，主要职责包含：

（1） 参与电池化成分容生产环节的软件研发工作，协助工程师完成相关软件模块的

设计、编码与测试。

（2） 参与项目需求分析，完成软件界面的原型绘制。

（3） 协助制定软件开发进度，确保项目按时按质完成。

（4） 负责下位机硬件的电路测试，编写和维护相关技术文档。

（5） 积极参与团队内部的技术讨论与交流，学习并应用新技术，提升个人技术水平。

（6） 配合团队完成其他相关工作，确保项目的顺利进行。二、实习目的及任务

1.实习目的

根据人才培养方案要求，学生在第八学期 1-7 周可以结合自己兴趣爱好与专业技术特

长完成创业型毕业实习或应用型毕业实习。

毕业实习目的是使学生巩固和运用所学的基础理论、基础知识和基本技能，获得有关

工作的实际锻炼，了解管理、研发和营销等工作的实际情况，培养在各种与专业相关工作

岗位上的独立工作能力，增强其毕业后从事相关工作的适应性。

2.实习任务

1、真正参与到企业相关的实际项目研发过程中，按时完成本人的任务；

2、学生自觉遵守企业规章制度，诚信做人，踏实做事，虚心学习；

3、综合运用所学专业知识获得独立工作的能力，强化自身的工程实践能力和创新能力，

开拓思维，掌握系统理念和管理技能，并培养较强的专业能力、团队协作能力、自主学习

能力、沟通表达能力、良好的适应发展的能力和良好的综合素质；

4、加深认识和领会党的方针政策，进一步提高思想觉悟，树立吃苦、耐劳的思想品德，

实习内容：

第一周的主要工作是熟悉公司的企业文化、主要业务和规章制度。

在企业文化方面，我深入了解了公司的核心价值观和发展理念 [1]。其中企业

规章制度是最为重要的，如同入学大学之后，需要熟知学生手册，了解学生规

章制度一样，企业的规章制度可以明确自己的职责和权利，了解企业的薪资构

成、加班福利制度、休假制度和应灾知识等，在更有效率地适应入职生活的同

时，也可以规避法律风险 [2]，避免因为不了解的情况下违反了公司的规定甚至做

出违法行为。

刚刚入职我的工作比较简单，开始几天便详细了解了公司的信息，参与了

公司内的入职会议，知道了哪些行为是公司的禁忌，避免了之后犯下错误的风

险，与此同时，除了公司总的规定，部门内组内也会有一些细节规定，如代码

的命名规范、共享文件的账号密码地址要求、项目更改的提交流程要求等，这

些规定与工作有更亲密的挂钩，是之前项目中积累的经验，在工作中避免犯规

定内的错，是个人业务能力的基本要求，因此也是需要熟知的。在学习结束之

后，会进行小的测试，验证是否熟知上文所述的知识。

除了熟悉公司的基本信息外，第一周还需要我开始上手熟悉自己的工作内

容，前期的工作内容比较轻松，并且有资历更深的同事领导帮助，入职到离职

期间部门内的项目进度都处于上位机阶段，所以第一周的代码或专业相关工作

基本上是在已经完成的模块中对细节再次优化，如记录部门内实际开发过程中

与 ui 设计的差异和需要更新的地方并向 ui 设计师进行反馈，这项任务也将持

续到我任职结束，其中我直观地感受到了企业与学校的差异，在学校中往往是

个人或同学组队进行小的项目开发，在制定需求、进行开发、优化完善、测试完工的过程中，往往是一个人进行多项工作，并且有修改的时候最多是口头或

信息阐述，不需要进行记录、审批，但在企业的工作中，为保证落实的项目是

最稳定、安全的一个版本，每个人的工作都被切割成了项目的一个部分 [3]，大家

在完成分内工作的同时，进行的更新、修改也需要上报审批，并且通知到关联

修改的同事一同修改，才能继续进行，否则将被打回。并且，这也是我第一次

与 ui 设计师打交道，初入职一周让我感到新奇且收益颇丰。

这次实习是我第一次接触详细的原型绘制，产品或数据系统的一个基本的

实用模型。它通常为示范目的或开发程序的部分结构而设计，是系统开发生命

周期（SDLC）中的一个关键阶段 [1]。

此前在学校的一些小项目中也会接触到原型绘制，但并不像工作中的考究

细节，往往确定好前端的交互逻辑是完整正确的便会进入开发，即使后续开发

过程中有出入，也不会回来修改原型，直接在代码处进行修改即可，而工作中

和技术的综合性行业。其基础是锂电池的制造，这是一种以锂离子嵌入和脱出

为基础的电池技术。在充放电过程中，锂离子在正负极材料之间来回迁移，实

现了电能的储存和释放 [2]。其中许多产业知识我并不了解，因此很多数据是不确

定甚至是不知道的，在这个过程中我经常需要请示领导，领导也悉心解释许多

数据的作用，属于生产的哪一个环节以及为什么归到某一类、为什么在原型的

某一页展示等，让我对参与的项目有了进一步的了解。在原型绘制的过程中，由于不太熟悉，因此初期效率较低，在被提醒后我

感觉到比较失落和紧张，担心自己因为业务能力不足而被辞退，开始在下班时

间进行额外的学习，所幸最后满足了工作要求，完成了自己的任务。

另外在与 ui 设计师沟通的任务中，发生了一件插曲让我更意识到了工作需

要谨慎。设计师提供的 ui 页面内的字体由于软件工程师的框架内并不包含该字

体源，因此无法使用该字体进行开发，领导让我就此事进行沟通，希望能换成

默认字体“微软雅黑”，会减少开发中的导包、兼容等琐碎问题。我与设计师

沟通后，她告知我使用“微软雅黑”会导致版权问题，该字体要求不能进行商

用，此时我才惊觉还有这个风险，因为在学校内大多数项目和系统并不会推广

到市场，因此很多时候并没有对字体、图片等进行规定，但这在企业中却是大

忌，由不同作者就同一题材创作的作品，作品的表达系‘独立完成’，并且具

有‘创作性’的，应当认定作者各自享有独立著作权 [3]，将作品私自商用，将违

反《著作权法》和《计算机软件保护条例》的相关规定 [4]。

后期寻找到了另外一款在开发框架内开源的字体进行替代，解决了这次的

问题。我也在这个过程中学习到了很多企业与学校环境的差异，后续在工作中

的开发需要更加谨慎才能避免经济、法律上的风险。

法来定义和程序逻辑分离的用户界面 [3]，对于这两种语言我都并不熟悉。为了能

在计划的工期内交差，我一边学习一边尝试地开发，期间也借用了一些工具，

2. 实习参与项目背景介绍

锂电池的生产是一个包含多项工序的流程，如图５所示。

图５ 锂电池生产流程图

前段生产工艺主要是进行电池材料的准备，中段生产工艺是将电池材料注入电

池的金属容器中并进行封装，此时电池还尚未具备充放电功能，需要去进行激发，

这也是后段工艺的作用。其中，化成便是处理组装后的电池，对其进行一定的电流处理，使得锂电池正

负极活性物质被激发，电池具备放电能力。化成结束后往往需要进行 OCV 测量，测

量电池的开路电压，最后再进行分容操作，分容操作是通过对锂电池的充电放电检

测，检测出每个电池的实际容量，并按照容量大小分为不同等级，最终进行各项指

标测试、打包，这便是锂电池生产的全过程。

电池化成分容系统是电池生产、分组过程中的关键设备之一，直接关系到电池

的质量和产量 [1]。许多动力电源电池需要以电池组的形式进行供电，如果电池组中某

一个电池容量不一致，导致存在过充或者过放的现象，非常容易造成整个电池组的

损坏。因此电池配组时需要充放电特性上有比较好的一致性 [2]，保证电池供能的稳定

与安全。

为了达到保证电池充放电特性上的一致性，后段生产工艺中，具备多种化成分

容方式，如恒压、恒流、恒功率等，在后段生产中也需要实时检测并打印相关参数，

往往还需要生成可视化的图像信息查看电池的各项指标趋势，对于不符合标准范围

内的电池，需要离开停止充放电，等待人工进行后续操作。

其中 SOC 和 SOH 是电池后段生产的两大重要指标 [3]。SOC 是指锂电池的荷电状态

指标，被定义为剩余容量与最大可用容量之比，用于量化当前电池内部所剩的能量，

准确的 SOC 能够防止电池的过充电或过放电，延长电池寿命 [4]。SOH 是指锂电池健康

状态，暂没有统一的定义，但通常以锂电池容量和内阻的变化来定义 [5]，可以用来描

述锂电池的老化情况。然而，SOC 和 SOH 均为锂电池状态的定义量，而并非具体的物

理量，无法被直接测量，往往需要依靠电池管理系统(BMS)通过设计的估算方法获取。

后段工艺流程中，对电池进行充放电操作的部件称为针床，针床按照电池的布

局和大小，引出了多个探针对接电池的正负极，当针床压下贴合电池托盘时，探针

便会接在电池正负极上，实现对电池的充电和放电，除此之外针床还有其他传感器

去检测电池的生产指标。在自动化流水线生产中，针床的下压和脱开都是生产文件

自动控制的，当发生一批电池中的部分电池停止工艺流程时，此时需要人为手动操

作针床。

实习参与项目为面向自动化和实验室的锂离子电池后段生产工艺的系统与解决

方案。锂离子电池自动化生产中包含许多操作与检测指标，部分指标需要上位机系

统进行估算，并且锂电池生产是比较危险的生产流程，为保证工人的安全，防止电池柜失火、爆炸威胁生命财产安全，使用上位机远程操作生产流程是可行方案。而

面向实验室的锂离子电池化成分容柜，往往是单个设备对于一批电池进行反复充电

放电并记录数据趋势，因此也需要上位机软件去接收、存储、解析数据，方便实验

人员的查看。综上所述，锂离子电池生产后段工艺的上位机、下位机系统具有可观

的前景。

制原型，其中一个模块往往包含多个页面，在原型绘制结束后，需要添加对应的跳

转交互等，保证模块内交互的逻辑正确，并且原有的原型具备进入新模块的入口，

新模块也具备出口返回原有的原型，此时原型绘制任务基本结束，需要经由领导审

核后再交付给 ui 设计师等待 ui 界面绘制，后续工作同更新原型类似。

（2） 模拟下位机的软件界面与逻辑开发

到实习中期，我对系统的了解已经比较深刻，开始接触一些代码相关工作。参

与项目的上位机软件开发使用的是 C#语言和 wpf 框架。

C#是微软公司发布的一种面向对象的编程语言，由 C 和 C++衍生而来。它运行

于.NET Framework 和.NET Core 之上，是一种高级程序设计语言。C#综合了 VB 简单

的可视化操作和 C++的高运行效率，以其强大的操作能力、优雅的语法风格、创新的

语言特性和便捷的面向组件编程的支持成为.NET 开发的首选语言，在市场中主要应用于游戏开发和制造业软件开发。

WPF（Windows Presentation Foundation）是微软推出的基于.NET Framework

的图形呈现系统，用于构建富客户端应用程序。它提供了统一的编程模型、语言和

框架，用于在桌面、Web 以及移动设备应用程序中构建用户界面。

图６ WPF 架构图

WPF 的架构图如图６所示。自顶向下，WPF 主要由 PresentationFramework、

PresentationCore、milcore 三层组件构成，其中 Common Language Runtime 是.Net

框架平台，DirectX 和 User32 都是 Windows 子系统 [6]。

在软件开发中，XML 作为界面描述语言在 20 世纪 90 年代中期变得极为流行 [7]。

微软提供了针对 Windows Vista/7 操作系统的专门界面开发语言 XAML，具备多样化

被编译成.Net 后台代码，因此 XAML 能够与.Net 语言如 C#进行协同工作，实现前端

界面和后台逻辑的同步开发，但依旧保证彼此的分离性，从而提高了开发效率，有

利于团队开发。

在后续查阅资料中，我了解到使用实习项目采用 wpf 框架的优点如下：

1) 面对锂离子电池生产行业，客户往往会使用 Windows 而非 Linux 系统进行操

作，对于非计算机行业的人员，Windows 作为主流的操作系统，更加熟悉且容易上手。

2) 以往下位机多采用 RS232 等串行通信接口标准与上位机 PC 进行连接，使用

wpf 即在.NET 环境下编程，集成了大量实用的类库，上位机部分也主要使用 SerialPort 类进行串口通信，采用 wpf 使得上位机软件编程部分大为轻松 [8]，也更好兼容

较早版本的下位机。

3) wpf 框架具备更丰富的界面，此类上位机系统此前已经有多家公司推出，但

由于是制造业软件，存在着界面不够美观，数据展示不够直观等缺点，因此在此方

面的优化需求下，使用 wpf 架构是最优选择。

（3） 上位机与下位机的通讯

上述后端逻辑开发中建立会话、实现通讯则使用到了 Socket 通讯模式，Socket的中文名称为套接字，是一种双向的通信端口，网络程序设计全靠套接字接受和发

送信息 [9]。Socket 的通讯流程如图７所示。

图７ Socket 通讯流程图

Socket 建立通讯主要分为以下步骤：

1) 服务器端创建一个 Socket 对象，并绑定到一个特定的端口上，然后开始监

听这个端口。

2) 客户端创建一个 Socket 对象，指明服务器的 IP 地址和端口号，并尝试连接。
3) 如果客户端能够成功连接，服务器端的监听将接收这个连接，并创建一个新

的 Socket 对象来处理与客户端的通信。

4) 客户端和服务器端通过使用Socket的发送(send)和接收(receive)方法来交

换数据。

5) 数据交换完成后，双方通过关闭 Socket 来结束通信。

因此在实际的开发中，后台获取到前台设定的 IP 地址和端口后，点击开始按钮

则触发 Socket 的创建，连接建立后通过 send 和 receive 方法进行通讯，最终通过结束按钮关闭 Socket 对象，结束连接。

本工作的也较为顺利，初期使用了本地的 IP 地址尝试是否传输成功，后期则部

署到不同的 PC 上进行测试，最终也圆满完成任务。
