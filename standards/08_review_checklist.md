## Python 基础层

## Python 结构设计层
- [ ] 核心领域规则没有长期漂移到多个 service / handler 中形成贫血模型风险
- [ ] 总线只用于一对多通知或跨边界协作，没有替代简单同步调用
- [ ] 对关键业务事件，没有默认吞异常继续
- [ ] registry 具备显式注册、重复检测、缺失报错
- [ ] 没有依赖裸全局 dict + 导入副作用完成关键注册
- [ ] 分发规则清晰，没有明显膨胀且难扩展的条件分支树
- [ ] 路由顺序没有让一般规则抢掉更具体规则
- [ ] 抽象层有真实替换需求、测试隔离价值或稳定边界
- [ ] 管线步骤输入输出清晰，且无隐藏共享状态
- [ ] 没有把大段杂糅逻辑伪装成“组合式管线”
