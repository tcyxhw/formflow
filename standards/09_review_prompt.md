# 规范审查 Prompt

请对当前任务涉及的代码进行规范审查，并按以下结构输出：

## 1. 合规检查
列出本次改动中：
- 遵守了哪些 stable 规则
- 哪些地方有风险但尚可接受
- 哪些地方违反了明确规则

## 2. 规则盲区
判断本次问题中是否出现了“规范未覆盖但重复可能性高”的模式。
只有满足以下条件才允许提议新规则：
1. 在真实代码中具有重复出现概率
2. 不遵守会带来明确风险
3. 10 秒内可理解
4. 不与 lint/type checker 重复

## 3. 规则瘦身建议
判断现有规则中是否有：
- 冗余
- 过度强制
- 与当前架构不符
- 应降级到 experimental 的内容

## 4. 输出格式
### 本次遵守
- ...

### 风险点
- ...

### 候选新规则
- 规则文本：
- 触发场景：
- 不遵守的后果：
- 适用范围：
- 例外场景：

### 建议降级/删除
- ...
