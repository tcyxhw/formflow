# 范式决策矩阵

## 核心决策表

| 场景特征 | 优先范式 | 关键工具 | 决策理由 | 禁忌 |
|:---|:---|:---|:---|:---|
| IO 密集(API/爬虫/文件网络) | 异步编程 | asyncio, httpx | 单线程高并发 | 禁止在 async 中调用同步阻塞 IO |
| CPU 密集(计算/压缩/加密) | 多进程 | ProcessPoolExecutor | 绕过 GIL | 禁止用多线程做 CPU 密集 |
| 大数据流(ETL/日志/大文件) | 生成器/流式 | yield, 迭代器 | O(1) 空间 | 禁止一次性加载大集合 |
| 核心业务(领域建模) | 类型化 OOP | Pydantic, dataclass | 强数据契约 | 禁止裸 Dict 跨边界 |
| 胶水脚本(一次性任务) | 函数式/轻量脚本 | 纯函数, 装饰器 | 开发效率优先 | 不要过度抽象 |
| 高频热路径 | 先测量再优化 | profile | 数据驱动 | 禁止未测量就优化 |

## 任务开始前必须回答
1. 这个任务的瓶颈是 IO 还是 CPU？
2. 数据规模是否可能超过内存？
3. 这是核心业务还是胶水层？
4. 数据要跨越几个边界？
